API Docs

Compact, AI-readable reference.

Every endpoint is shown with purpose, request shape, response shape, and one simple example.

Open Control UI

Base URL

http://localhost:3000

Flow

1. POST /api/token with x-admin-secret
2. Receive token scoped to db_name + access
3. POST /api/sql with Authorization: Bearer <token>
4. Review /api/logs for recent access records

GET /api/health

Returns service status and configured database names.

{
  "ok": true,
  "databases": ["main", "analytics"]
}

GET /api/databases

Returns the list of available db_name values.

{
  "databases": ["main", "analytics"]
}

POST /api/token

Creates a short-lived token for exactly one database and one access mode.

Headers:
  x-admin-secret: YOUR_ADMIN_SECRET

Body:
{
  "db_name": "main",
  "access": "read_only",
  "ttl_seconds": 3600
}
Success:
{
  "db_name": "main",
  "access": "read_only",
  "token": "<jwt-like-string>",
  "expiresInSeconds": 3600
}

POST /api/sql

Runs SQL against the database named in db_name. The token must match the same db_name.

Headers:
  Authorization: Bearer TOKEN

Body:
{
  "db_name": "main",
  "sql": "select now() as now",
  "params": []
}
Success:
{
  "db_name": "main",
  "access": "read_only",
  "command": "SELECT",
  "rowCount": 1,
  "rows": [
    { "now": "2026-04-21T00:00:00.000Z" }
  ]
}

GET /api/logs?limit=50

Returns recent log entries from logs/access.log.

{
  "logs": [
    {
      "time": "2026-04-21T10:00:00.000Z",
      "type": "token_issued",
      "db_name": "main",
      "access": "read_only"
    }
  ]
}

Rules

- token is bound to one db_name
- access is either read_only or full
- read_only accepts only SQL starting with SELECT
- full allows any SQL
- max token lifetime is 3600 seconds

Config

PORT=3000
PROXY_ADMIN_SECRET=change-this-admin-secret
PROXY_SIGNING_SECRET=change-this-signing-secret
POSTGRES_DATABASES={"main":"postgres://...","analytics":"postgres://..."}
