API Docs

Internal DB stores target DBs. Tokens use saved `db_name` values.

`DATABASE_URL` is the proxy app's own database. AI target databases are added through the admin API or the web UI.

Open Control UI

Base URL

http://localhost:3000

Flow

1. Set DATABASE_URL for the proxy app itself
2. POST /api/managed-databases with x-admin-secret
3. POST /api/token for one saved db_name
4. POST /api/sql with Authorization: Bearer <token>
5. Review /api/logs for access records

GET /api/health

Returns service health and number of saved target databases.

{
  "ok": true,
  "database_count": 2
}

GET /api/databases

Returns the list of saved db_name values.

{
  "databases": ["main", "analytics"]
}

GET /api/managed-databases

Admin-only. Returns saved target databases including connection strings.

Headers:
  x-admin-secret: YOUR_ADMIN_SECRET

Success:
{
  "databases": [
    {
      "db_name": "main",
      "connection_string": "postgres://user:password@host:5432/db",
      "created_at": "2026-04-21T01:00:00.000Z",
      "updated_at": "2026-04-21T01:05:00.000Z"
    }
  ]
}

POST /api/managed-databases

Admin-only. Creates or updates a target database that the AI can later use.

Headers:
  x-admin-secret: YOUR_ADMIN_SECRET

Body:
{
  "db_name": "main",
  "connection_string": "postgres://user:password@host:5432/db"
}

POST /api/managed-databases/test

Admin-only. Tests a candidate connection string before saving it.

Headers:
  x-admin-secret: YOUR_ADMIN_SECRET

Body:
{
  "connection_string": "postgres://user:password@host:5432/db"
}

DELETE /api/managed-databases/:db_name

Admin-only. Removes a saved target database.

POST /api/token

Admin-only. Creates a token for one saved db_name.

Headers:
  x-admin-secret: YOUR_ADMIN_SECRET

Body:
{
  "db_name": "main",
  "access": "read_only",
  "ttl_seconds": 3600
}
Success:
{
  "db_name": "main",
  "access": "read_only",
  "token": "<jwt-like-string>",
  "expiresInSeconds": 3600,
  "proxyUrl": "https://pg-proxy-production.up.railway.app/",
  "apiDocsUrl": "https://pg-proxy-production.up.railway.app/docs",
  "sqlUrl": "https://pg-proxy-production.up.railway.app/api/sql",
  "authorization": "Bearer <jwt-like-string>",
  "aiConnectionPacket": "POSTGRES PROXY CONNECTION\n..."
}

POST /api/sql

Runs SQL against the saved target database named in db_name.

Headers:
  Authorization: Bearer TOKEN

Body:
{
  "db_name": "main",
  "sql": "select now() as now",
  "params": []
}

GET /api/logs?limit=50

Returns recent log entries from logs/access.log.

Required Env

DATABASE_URL=postgres://proxy-app-db
PROXY_ADMIN_SECRET=change-this-admin-secret
PROXY_SIGNING_SECRET=change-this-signing-secret
PROXY_TOKEN_DEFAULT_TTL_SECONDS=3600
PUBLIC_PROXY_BASE_URL=https://pg-proxy-production.up.railway.app/

Rules

- DATABASE_URL is the proxy app's own database
- target databases are stored in managed_databases
- token is bound to one db_name
- access is read_only or full
- read_only accepts only SQL starting with SELECT
- ttl_seconds defaults to PROXY_TOKEN_DEFAULT_TTL_SECONDS
- set PROXY_TOKEN_MAX_TTL_SECONDS only if you want a cap
- token responses include a paste-ready AI connection packet
